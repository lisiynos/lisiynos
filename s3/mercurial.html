Система управления версиями Mercurail (hg)

Mercurial - распределённая система управления версиями написанная на Python'е. Она проигрывает по производительности git, но зато клиенты для mercurial лучше интегрированы в Windows, и легко найти бесплатный хостинг любого объёма (например, сайт bitbucket.org предоставляет любое количество открытых и закрытых репозитариев при условии что количество участников проекта не больше 5).

Скачивание и установка:

  • Клиент TortoiseHG для Mercurial для Windows: http://tortoisehg.bitbucket.org, скачайте и установите последнюю версию.
  • Под Linux: sudo apt-get install mercurial

 

Главный недостаток ручного тестирования: его нужно выполнять каждый раз когда программа меняется,  даже если изменился кусок кода от другого разработчика, наша часть могла перестать работать. При росте размера проекта количество практически одновременных изменений в различных кусках кода непрерывно возрастает, и количество тестов, которые нужно сделать чтобы убедиться что “мы ничего не сломали”  тоже возрастает. Начиная с некоторого объема проекта такое ручное тестирование становится слишком трудоёмким,  а следовательно, этим тестированием  пожертвуют в первую очередь и не будут выполнять. Именно по этой причине лучше сразу же делать модульные тесты.

Системы управления версиями*

Управление версиями также тем актуальнее, чем больше и сложнее проект. Для решения олимпиадных задач оно не актуально, так что эта часть общеразвивающая.

Централизованные: CVS, SVN - сихронизация (обмен исходными текстами программ) между разработчиками происходит через единый центральный сервер.
Децентрализованные: GIT, Mercurial, Bazaar - у каждого разработчика свой репозиторий с историей изменений, он может свободно обмениваться накопленными изменениями с любым другим разработчиком.
